# MirrorKit v1.1 Visual Customization — Manual Test Plan

**Pre-requisites:**
- macOS 14.0+ Sonoma machine
- Physical iPhone running iOS 17+ paired via USB
- A Release build of MirrorKit v1.1 from the worktree (or Debug build for in-development testing)

## Reset between runs

Before each clean run:
```bash
defaults delete com.achraftrabelsi.MirrorKit
```

## A. Background expanded mode

- [ ] First launch with no prior preferences → expanded mode shows the Midnight gradient (matches v1.0)
- [ ] Click each preset (Midnight, Charcoal, Snow, Sunset, Ocean, Aurora, Black) → expanded mode reflects the choice in real time
- [ ] Click "Custom…" → NSColorPanel opens, dragging changes the cell preview AND the expanded background live
- [ ] Quit and re-launch the app → preset is preserved
- [ ] Manually corrupt the hex value: `defaults write com.achraftrabelsi.MirrorKit backgroundCustomColor -string "garbage"` then launch → custom cell shows black, app does not crash
- [ ] Snow background with Floating bezel + drop shadow → the shadow is visible and not muddy

## B. Bezel customizable

- [ ] First launch with no prior preferences → Classic bezel (matches v1.0 default)
- [ ] Set legacy `defaults write com.achraftrabelsi.MirrorKit showDeviceFrame -bool false`, delete bezelStyle key, relaunch → app shows Frameless (migration worked)
- [ ] Set legacy `defaults write com.achraftrabelsi.MirrorKit showDeviceFrame -bool true`, delete bezelStyle key, relaunch → app shows Classic (migration worked)
- [ ] Switch through Classic / Floating / Frameless in Settings → mirror window updates in real time
- [ ] In Classic, switch Black / Silver / Gold → frame color updates in real time
- [ ] In Floating or Frameless, the Bezel color row is hidden in Settings
- [ ] Rotate the iPhone (Cmd+R or rotate button) → bezel rotates correctly in all three styles
- [ ] Floating shadow looks acceptable on Snow background (light) and Midnight background (dark)

## C. Annotations live

- [ ] Press hotkey `A` with mirror window focused → annotation mode toggles
- [ ] Click the pencil button in the toolbar → annotation mode toggles (same behavior as hotkey)
- [ ] Pen tool draws free-hand lines that follow the cursor
- [ ] Arrow tool draws a line with arrowhead from drag start to drag end
- [ ] Circle tool draws an ellipse fitting the bounding box of the drag
- [ ] Rectangle tool draws a rounded rectangle fitting the bounding box of the drag
- [ ] Highlighter tool draws a thick translucent stroke
- [ ] Each color (red/blue/green/yellow/white/black) applies on the next stroke
- [ ] The ↶ button reverts the last stroke (Cmd+Z is a v1.2 enhancement, not in this release)
- [ ] 🗑 (clear) removes all strokes; pressing undo right after restores them
- [ ] Switch device (in multi-device picker) → annotations clear, annotation mode turns off
- [ ] Disconnect iPhone → annotations clear
- [ ] Resize the window with strokes present → strokes scale correctly with the captureView (normalized coords)
- [ ] Rotate iPhone (capture resolution changes) → annotations clear (because the underlying content has rotated)
- [ ] Toggle annotation mode off → existing strokes stay on screen, drawing is disabled, zoom/pan still work
- [ ] Take a snapshot with strokes → saved PNG includes the strokes
- [ ] Take a snapshot without strokes → saved PNG is the raw mirror frame
- [ ] Take a video recording with strokes → strokes are NOT in the recorded mp4 (by design v1.1)

## D. App Store / sandbox sanity

- [ ] `xcodebuild archive` produces a Release build with no entitlement errors
- [ ] Open the archived `.app` directly → launches without any Gatekeeper / sandbox alert
- [ ] Run `nm -u <archived-binary> | grep -i 'mobile\|private'` → no private API symbols
- [ ] All `MirrorKitTests` pass via `xcodebuild test`
